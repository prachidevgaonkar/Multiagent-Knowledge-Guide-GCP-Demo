What I Learned

Non-obvious lessons from building this

Documented here so you don't have to discover them the hard way.


🎯
Start with one agent, not many
Built three sub-agents on day one. Should have started with one and split only when context switching demonstrably failed. Routing errors compound.
Agent Design
📝
The system prompt is the product
Response quality is almost entirely determined by the system prompt. Version it, review it, test it — treat it like production code.
Agent Design
🌡️
Temperature=0 for structured data
NL-to-SQL at temperature > 0 produces non-deterministic SQL. The same query can return different results across sessions. This was the single highest-impact fix.
Issue Agent
⏱
Async is not optional
Chat platforms expect fast acknowledgement. Multi-source agent calls take longer. A durable task queue is the only reliable separation between UX and processing.
Integration
👁
Freshness is invisible without signals
If a doc is updated and not re-indexed, the agent answers from the old version with full confidence. Freshness timestamps must be surfaced explicitly.
Data Sources
🗺
Examples matter as much as rules
"How does authentication work?" was mis-routed to the Code agent. Adding disambiguation examples to the orchestrator prompt fixed routing more than rewriting the rules.
Routing

Active Issues

Open questions — still learning

These are the active learning surface of the POC, not bugs to block launch.


#1
Inconsistent responses on issue data
Same question can return different counts across sessions when the data hasn't changed. Root cause: NL-to-SQL non-determinism at temperature > 0. Fixed with temperature=0 and curated BigQuery views. SQL validation layer and schema auto-sync still planned.
Active
#2
Delayed or missing chat responses
Longer analytical queries can exceed the chat platform's acknowledgement window. Background threads also fail when ephemeral function instances end early. Fixed immediate acknowledgement and failure callbacks; managed task queue still in progress.
Active
#3
Data and code freshness lag
Periodic exports and indexing introduce a delay between source updates and searchable content. The agent may answer from a stale snapshot with no visible warning. Incremental BigQuery updates and freshness indicators in responses are planned.

Strategic Roadmap

Where this goes next

Capability areas for the next phase — from text assistant to adaptive, multimodal experience.


🎨
Multimodal outputs
Visual flows, diagrams, spoken summaries, short video walkthroughs, and structured briefing documents — the right format for the right question.
🎙
Voice interaction
Speak naturally, receive spoken or visual responses. Hands-free information retrieval, accessible onboarding, and field-based workflows.
📊
Continuous evaluation
Representative test sets for all user types. Measure groundedness, routing quality, safety, latency, and cost. Regression evals whenever prompts or models change.
🔄
Governed self-improvement
Analyze failures, propose prompt revisions, test in sandbox, submit for human approval before deploying. Observe → diagnose → propose → evaluate → approve → deploy → monitor.
